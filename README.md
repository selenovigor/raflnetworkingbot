# Telegram bot for comments under one post

Бот для канала `@rafl_work`: отвечает в комментариях только под одним заданным постом и только реплаем на сообщение пользователя.

## Стек

- Python 3.9+
- aiogram 3
- БД не нужна: логика статичная, состояние хранить не требуется
- Хостинг: Render или Railway как самый простой старт; в проекте есть `Procfile`

## Как это работает

Комментарии к посту канала Telegram приходят боту как сообщения в привязанной группе обсуждений. Поэтому бот должен быть добавлен не только в канал, а именно в группу обсуждений канала.

Фильтр сделан по `POST_ID`: бот проверяет, что сообщение относится к нужному корневому посту/треду, и игнорирует всё остальное.

## Подготовка

1. Создать бота через [@BotFather](https://t.me/BotFather) и получить `BOT_TOKEN`.
2. Добавить бота в тестовый канал и в привязанную тестовую группу обсуждений.
3. В BotFather отключить privacy mode для бота:
   - `/setprivacy`
   - выбрать бота
   - `Disable`
4. Опубликовать тестовый пост с картинкой.
5. Открыть комментарии к посту и определить `POST_ID` корневого сообщения в группе обсуждений.

Если `POST_ID` непонятен, на тесте включить в `.env`:

```bash
DEBUG_UPDATES=true
```

После комментария под тестовым постом бот выведет в консоль:

```text
message chat_id=-100... message_id=... thread_id=... reply_to=... text='1'
```

Для `TARGET_CHAT_ID` берём `chat_id`. Для `POST_ID` обычно подходит `thread_id`; если его нет, берём `reply_to`.

## Локальный запуск

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
```

Заполнить `.env`, затем запустить:

```bash
set -a
source .env
set +a
python bot.py
```

## Тестовый прогон

В тестовом канале:

1. Опубликовать пост с картинкой.
2. Открыть комментарии.
3. Написать `1`, `2`, `3`, `4`, `5` отдельными комментариями.
4. Проверить, что бот отвечает реплаем на каждый комментарий.
5. Написать любой другой текст, например `футбол`.
6. Проверить отбивку:

```text
Здесь работают только цифры
Посмотри на картинку и напиши номер — от 1 до 5
```

7. Написать сообщение под другим постом и убедиться, что бот молчит.

## Деплой

Для Render/Railway:

1. Загрузить проект в GitHub.
2. Создать worker/background service.
3. Build command: `pip install -r requirements.txt`.
4. Start command: `python bot.py` или использовать `Procfile`.
5. В переменные окружения добавить `BOT_TOKEN`, `POST_ID`, `TARGET_CHAT_ID`.

## Переменные окружения

`BOT_TOKEN` — токен бота от BotFather.

`POST_ID` — ID корневого сообщения/треда в группе обсуждений под конкретным постом.

`TARGET_CHAT_ID` — ID группы обсуждений. Необязательно, но рекомендуется, чтобы бот не реагировал в других чатах.

`LOG_LEVEL` — уровень логов, по умолчанию `INFO`.

`DEBUG_UPDATES` — выводит ID входящих сообщений для настройки тестового канала.
